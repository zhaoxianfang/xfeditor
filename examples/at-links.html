@链接 和 Email 自动链接

Github Flavored Markdown — @提及 与 自动邮箱链接 功能演示与详解

📖 功能说明

xfEditor v1.17.8 默认启用了 @link 和 Email 自动链接 功能：

@ 用户提及 将形如 @用户名 的文本自动转换为链接，指向 GitHub（或其他自定义平台）的用户主页
Email 自动链接 将形如 user@domain.com 的邮箱地址自动转换为 mailto: 链接

🔧 配置方式

基础 URL 配置 — 在初始化前修改全局变量：

editormd.urls.atLinkBase = "https://github.com/"; // @link 的基础 URL

禁用功能 — 在配置中关闭：

testEditor = editormd("test-editormd", {
    atLink    : false,    // 禁用 @link 自动链接
    emailLink : false,    // 禁用 email 地址自动链接
    path      : '../lib/'
});

🎯 语法规则

语法格式匹配示例渲染结果
@username@pandao<a href="https://github.com/zhaoxianfang">@pandao</a>
@username@tylerlong<a href="https://github.com/tylerlong">@tylerlong</a>
邮箱自动链接test@gmail.com<a href="mailto:test@gmail.com">test@gmail.com</a>

⚠️ 注意事项

  1. 标准 email 地址中的 @ 不会被混淆（如 xxx.test@gmail.com 中的 @ 会正确分为邮箱部分和 @link 部分）
  2. 邮件格式 <xxx.test@gmail.com> 或 <mailto:xxx.test@gmail.com> 也会自动识别
  3. Markdown 链接中的 @ 不会被额外处理（如 [Gmail](mailto:xxx@gmail.com)）
  4. HTML 标签内的 @ 不会被处理，避免破坏标签结构
  5. 通过 atLink: false 可完全禁用此功能
