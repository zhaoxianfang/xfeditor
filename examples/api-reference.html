快速开始

基本初始化

// 最简初始化（仅 3 个必填参数）
var editor = xfEditor("editor-id", {
    width  : "100%",    // 宽度，默认 "100%"
    height : 640,       // 高度，默认 "100%"
    path   : "./lib/"   // 依赖模块目录，默认 "./lib/"
});

第一个参数为容器 ID（该容器的直接子元素为 <textarea>），第二个参数为配置对象，返回编辑器实例。

HTML 结构要求

<!-- 容器 ID 必须与 xfEditor() 第一个参数一致 -->
<div id="editor-id">
    <textarea>初始 Markdown 内容（可选）</textarea>
</div>

<!-- 引入依赖（位置建议在 </body> 之前） -->
<link rel="stylesheet" href="css/xf_editor.min.css" />
<script src="js/jquery.min.js"></script>
<script src="xf_editor.min.js"></script>

工具栏 Object 类型参数详细说明

toolbarTitles（Object 类型） — 自定义按钮的 hover 提示文本。Key 为按钮名称，Value 为提示字符串。默认值 {}（使用语言包中的默认文本）。
// 示例：自定义部分按钮的提示文本
toolbarTitles: {
    bold    : "加粗 (Ctrl+B)",
    image   : "上传图片",
    preview : "预览切换"
}
toolbarHandlers（Object 类型） — 自定义按钮的点击处理函数。Key 为按钮名称，Value 为 Function。默认值 {}。函数中的 this 指向编辑器实例。
// 示例：给 bold 按钮添加自定义逻辑
toolbarHandlers: {
    bold: function() {
        // this = 编辑器实例
        var cm = this.cm;
        var cursor = cm.getCursor();
        var selection = cm.getSelection();
        cm.replaceSelection("**" + selection + "**");
        this.save(); // 手动触发保存/刷新
    }
}
toolbarCustomIcons（Object 类型） — 自定义按钮的完整 HTML 结构。Key 为按钮名称，Value 为 HTML 字符串。默认值内置了 lowercase、ucwords、formula 三个按钮的自定义 HTML。
// 示例：完全自定义一个按钮的 HTML
toolbarCustomIcons: {
    myButton: '<a href="javascript:;" title="我的按钮"><i class="fa fa-star"></i></a>'
}
toolbarIconsClass（Object 类型） — 自定义按钮的 CSS 类名。Key 为按钮名称，Value 为 CSS 类名字符串。默认值内置了 55+ 按钮的 Font Awesome 图标类名映射。
// 示例：将 bold 按钮图标改为自定义图标
toolbarIconsClass: {
    bold  : "my-custom-bold-icon",
    image : "fa fa-cloud-upload"
}
toolbarIconTexts（Object 类型） — 自定义按钮的显示文本（替换图标）。Key 为按钮名称，Value 为文本字符串。默认值 {}。
// 示例：用文字替代图标显示

    bold  : "B",
    image : "图"

htmlDecode 参数详解 安全关键

htmlDecode 是安全敏感参数，控制 Markdown 中内嵌 HTML 的解析行为。无论配置为何值，XSS 安全过滤始终生效（危险标签、事件属性、危险协议等自动移除）。
一、基本取值模式
值行为说明与示例
false不解析 HTML 默认值、最安全。所有 HTML 标签原样显示为文本，<script>、<iframe> 等危险标签以纯文本显示。
htmlDecode: false → 输入 <b>加粗</b> 显示为 &lt;b&gt;加粗&lt;/b&gt;（纯文本，Marked 自动转义）
true解析全部 HTML 允许所有 HTML 标签被 Marked 解析并渲染为真实 HTML 元素。注意：即使设为 true，系统内置 XSS 安全过滤会自动移除 <script>、<iframe>、<form>、<object> 等危险标签，以及 on* 事件属性（如 onclick、onerror）、javascript:/vbscript: 协议等。
htmlDecode: true → 输入 <b>加粗</b> 显示为 加粗
"tag1,tag2,…"黑名单模式 允许 HTML 解析，但移除列表中指定的标签（黑名单），其余标签正常渲染。
htmlDecode: "span,div" → <span>和<div>标签被移除，其余标签（如 <b>、<i>、<table>）正常显示
二、标签 + 属性过滤规则（格式："标签列表|属性规则"）
语法格式：htmlDecode: "要移除的标签,逗号分隔|属性过滤规则"
用 | 分隔左右两部分——左侧是要移除的标签（黑名单），右侧是剩余标签的属性过滤规则。
值左侧：标签过滤右侧：属性过滤规则效果说明
"style,script,iframe" 移除 style、script、iframe 标签 （无，省略了 | 及右侧） 仅移除这三个标签，其余标签的属性和内容完整保留。适合允许用户使用 HTML 但又担心内联样式注入的场景。
"style,script,iframe|*" 移除 style、script、iframe 标签 * → 剥离所有属性 移除三个标签后，对其余所有标签清除全部属性（class、id、style、href 等全部删除），只保留标签名和文本内容。这是最强安全策略之一。
"style,script,iframe|on*" 移除 style、script、iframe 标签 on* → 剥离事件属性 移除三个标签后，对其余所有标签只清除 on 开头的事件属性（onclick、onerror、onload、onmouseover…），保留 class、id、style、href 等正常属性。
"script,iframe|style,class" 移除 script、iframe 标签 style,class → 剥离指定属性 移除 script/iframe 后，对其余标签清除 style 和 class 属性，其余属性保留。适合需要布局控制但禁止自定义样式的场景。
"script|id,data-*" 移除 script 标签 id,data-* → 剥离指定属性 移除 script 标签后，对其余标签清除 id 和 data-* 属性，其余属性保留。
三、属性过滤规则详解
规则值作用详细说明
* 清除所有属性 对所有未被黑名单移除的标签，剥离其全部 HTML 属性（class、id、style、href、src、alt、title 等），只保留纯标签名和文本内容。
例如 <a href="https://x.com" class="link" target="_blank">链接</a> 将变为 <a>链接</a>
on* 清除事件属性 对所有标签清除 以 on 开头 的事件处理属性。包括但不限于：onclick、onerror、onload、onmouseover、onmouseout、onfocus、onblur、onchange、onsubmit、onkeydown、onkeyup、onkeypress、ondblclick、onmousedown、onmouseup 等。
这是 XSS 防御的关键规则——即使系统已自动移除 on*，再次使用此规则可提供深度防御。
"attr1,attr2,…" 清除指定属性 对所有标签清除逗号分隔的指定属性名。例如 "style" 清除所有 style 属性，"style,class" 同时清除 style 和 class，"id,data-url" 清除 id 和 data-url。
四、实战配置示例
// ===== 场景一：完全关闭 HTML（默认、最安全） =====
htmlDecode: false

// ===== 场景二：完全信任（解析所有 HTML，XSS 过滤仍生效） =====
htmlDecode: true

// ===== 场景三：移除危险标签，保留所有属性 =====
// 适合用户评论/UGC 场景，担心 style 注入 CSS 攻击
htmlDecode: "style,script,iframe"

// ===== 场景四：移除危险标签 + 清除所有属性（极严格） =====
// 适合对安全要求极高的公开内容平台
htmlDecode: "style,script,iframe|*"

// ===== 场景五：移除危险标签 + 清除事件属性（推荐生产环境） =====
// 保留 class/style/href 等正常属性，仅移除事件处理器
htmlDecode: "style,script,iframe|on*"

// ===== 场景六：移除危险标签 + 清除 class/style（控制外观） =====
// 允许用户使用 HTML 结构但禁止自定义样式
htmlDecode: "script,iframe,object|style,class"

// ===== 场景七：仅限制最核心危险标签，其余自由 =====
htmlDecode: "script"

// ===== 场景八：用户内容需要表格排版 =====
// 移除 script/iframe，剥离所有属性（防止注入），保留表格标签
htmlDecode: "script,iframe|*"
重要提示：
  • 黑名单语义：字符串中的标签列表是黑名单（要移除的），而非白名单。例如 "script,iframe" 表示"移除 script 和 iframe"，而不是"只允许 script 和 iframe"。
  • XSS 安全过滤始终生效：无论 htmlDecode 配置为何值，系统内置的危险标签（script、iframe、frame、object、embed、applet、form、base、link、meta 等 20+ 种）、危险事件属性（onclick、onerror 等 60+ 种）和危险协议（javascript:、vbscript:、data:text/html 等）都会被自动过滤。
  • 标签移除优先级：XSS 安全过滤 → 用户标签过滤 → 属性规则过滤，按此顺序执行。
  • 属性规则仅影响黑名单移除后剩余的标签：已被黑名单移除的标签不会经过属性规则处理。
  • true ≠ 完全开放：即使设为 true，危险标签和属性仍会被系统自动移除。
